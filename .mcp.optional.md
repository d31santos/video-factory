# Optional MCP servers (not wired by default)

The v5 inventory lists two optional asset servers. They are **not** in `.mcp.json`
because their exact package names weren't verified here and they need API keys — add
them yourself once verified, following the `pnpm dlx` + env-var pattern:

```jsonc
"pexels": {
  "command": "pnpm",
  "args": ["dlx", "pexels-mcp"],          // verify package name on npm first
  "env": { "PEXELS_API_KEY": "${PEXELS_API_KEY}" }
},
"replicate": {
  "command": "pnpm",
  "args": ["dlx", "replicate-mcp"],        // verify package name on npm first
  "env": { "REPLICATE_API_TOKEN": "${REPLICATE_API_TOKEN}" }
}
```

- **Pexels** — free stock B-roll (Pexels API key is free). Drop clips into `assets/broll/`.
- **Replicate** — generated images/clips; set a monthly spend cap. Costs scale with use.

## Notes on the wired servers
- **remotion** — no credential.
- **descript** — needs `DESCRIPT_API_TOKEN` (dev portal, scoped to a dedicated Drive) in the environment.
- **opusclip** — OAuth, **not** an API key: a browser window opens on the first tool call to sign in
  to your OpusClip account and approve access. Hosted MCP (`mcp.opus.pro`). Connect your
  target platforms (YouTube/IG/TikTok/LinkedIn/X) *inside* OpusClip — credentials never touch this repo.
